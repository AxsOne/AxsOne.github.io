RFC for Game Networking Packets

This document describes the packet structure for game networking, focusing on the data exchanged between clients and servers using both TCP and UDP protocols.

Packet Name Field Type Description
RequestTCPUDPLinkPacket type PacketType_t Specifies the packet type, indicating that the client is requesting a link between TCP and UDP.
ResponseTCPUDPLinkPacket type PacketType_t Specifies the packet type, used to respond to a TCP-UDP link request.
udpPort unsigned short The UDP port the server is listening on.
tcpPort unsigned short The TCP port the server is listening on.
ip char[16] The server's IP address, formatted as a string (IPv4).
CreateGameRoomPacket type PacketType_t Specifies the packet type, indicating a request to create a new game room.
roomName char[256] The desired name for the game room.
ip char[16] The client's IP address, formatted as a string (IPv4).
JoinGameRoomPacket type PacketType_t Specifies the packet type, indicating a request to join an existing game room.
roomName char[256] The name of the game room to join.
ip char[16] The client's IP address, formatted as a string (IPv4).
RoomNotFoundPacket type PacketType_t Specifies the packet type, indicating that the requested game room was not found.
StartGamePacket type PacketType_t Specifies the packet type, indicating a request to start the game in a specific room.
roomName char[256] The name of the game room where the game should start.
LoadPacket type PacketType_t Specifies the packet type, indicating that the server is loading a level.
level int The level number being loaded.
player_count int Number of players in the game.
player_id int The ID of the player this packet is intended for.
speed_x float Initial horizontal speed for the player.
speed_y float Initial vertical speed for the player.
LoadReadyPacket type PacketType_t Specifies the packet type, indicating that the client is ready after receiving the load data.
GoPacket type PacketType_t Specifies the packet type, used to signal the start of the game or a specific phase.
MovePacket type PacketType_t Specifies the packet type, indicating a movement action from a client.
entity_id int The ID of the entity that is moving.
speed_x double The horizontal speed of the entity.
speed_y double The vertical speed of the entity.
AuthorativeMovePacket type PacketType_t Specifies the packet type, used by the server to dictate an entity's movement.
entity_id int The ID of the entity whose movement is being directed.
speed_x double The server-defined horizontal speed of the entity.
speed_y double The server-defined vertical speed of the entity.
AuthorativePositionPacket type PacketType_t Specifies the packet type, used by the server to update an entity's position.
entity_id int The ID of the entity whose position is being updated.
x double The updated X-coordinate of the entity.
y double The updated Y-coordinate of the entity.
StartShootPacket type PacketType_t Specifies the packet type, indicating a request to start shooting.
entity_id int The ID of the entity starting to shoot.
StopShootPacket type PacketType_t Specifies the packet type, indicating a request to stop shooting.
entity_id int The ID of the entity stopping to shoot.
AuthorativeShootPacket type PacketType_t Specifies the packet type, used by the server to notify about a shot fired.
entity_id int The ID of the entity that fired the shot.
bullet_id int A unique ID for the bullet.
x double The X-coordinate where the bullet was spawned.
y double The Y-coordinate where the bullet was spawned.
velocity_x double The horizontal velocity of the bullet.
velocity_y double The vertical velocity of the bullet.
